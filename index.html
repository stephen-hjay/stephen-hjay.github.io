Posted on 2020-05-13

JSON：JavaScript Object Notation

概念： JavaScript Object Notation JavaScript对象表示法

​ var p = {“name”:”张三”,”age”:23,”gender”:”男”};

  • json现在多用于存储和交换文本信息的语法

进行数据的传输

  • JSON 比 XML 更小、更快，更易解析。

语法：

  • 数据在名称/值对中：json数据是由键值对构成的
  • 键用引号(单双都行)引起来，也可以不使用引号
  • 值得取值类型：
    1. 数字（整数或浮点数）
    2. 字符串（在双引号中）
    3. 逻辑值（true 或 false）
    4. 数组（在方括号中） {“persons”:[{},{}]}
    5. 对象（在花括号中） {“address”:{“province”：”陕西”….}} json中的json对象
    6. null
  • 数据由逗号分隔：多个键值对由逗号分隔
  • 花括号保存对象：使用{}定义json 格式。就是 Json本质上就是一种对象字典
  • 方括号保存数组：[]

获取数据:

  1. json对象.键名（注意是直接打键名）
  2. json对象[“键名”] 适用于遍历键名
  3. 数组对象[索引]

JSON数据和Java对象的相互转换

服务器需要将json的数据 转换为Java对象进行利用。

服务器从客户端（浏览器）获取json数据，并将其转换为Java对象进行数据获取，服务器将计算得到的Java对象 转换为json数据 返回给浏览器端进行解析。

  • JSON解析器：
    • 常见的解析器：Jsonlib（官方提供的），Gson（Google提供的），fastjson（阿里巴巴提供的），jackson（springMVC 内置解析器）

本质就是javascript字典与JavaBean (JavaBean数组 或者是Map) 之间的转换

JSON转为Java对象

  1. 导入jackson的相关jar包
  2. 创建Jackson核心对象 ObjectMapper
  3. 调用ObjectMapper的相关方法进行转换
    1. readValue(json字符串数据, JavaBean.Class)

Java对象转换JSON

  1. 导入jackson的相关jar包

  2. 创建Jackson核心对象 ObjectMapper

  3. 调用ObjectMapper的相关方法进行转换

    有2种主要的方法：

    • writeValue(参数1，obj):
      File：将obj对象转换为JSON字符串，并保存到指定的文件中
      Writer：将obj对象转换为JSON字符串，并将json数据填充到字符输出流中
      OutputStream：将obj对象转换为JSON字符串，并将json数据填充到字节输出流中
    • writeValueAsString(obj):将对象转为json字符串
2. 注解：加在JavaBean的字段上
 3. @JsonIgnore：排除属性。
 4. @JsonFormat：属性值得格式化

  • @JsonFormat(pattern = “yyyy-MM-dd”)

    1. 复杂java对象转换
    2. List：数组 可以直接转换
    1. Map：对象格式一致 就是一个JavaBean

JavaScript|Ajax异步请求

Posted on 2020-05-13

AJAX：

概念： ASynchronous JavaScript And XML 异步的JavaScript 和 XML

  1. 异步和同步：客户端和服务器端相互通信的基础上
    • 客户端必须等待服务器端的响应。在等待的期间客户端不能做其他操作。
    • 客户端不需要等待服务器端的响应。在服务器处理请求的过程中，客户端可以进行其他的操作。

Ajax 是一种在无需重新加载整个网页的情况下，能够更新部分网页的技术。 [1] 通过在后台与服务器进行少量数据交换，Ajax 可以使网页实现异步更新。这意味着可以在不重新加载整个网页的情况下，对网页的某部分进行更新。传统的网页（不使用 Ajax）如果需要更新内容，必须重载整个网页页面。提升用户的体验

同步和异步的区别：（有点类似于多线程的概念）

1、同步：客户端当发出请求的时候，一直在等待服务器的响应。期间不能做任何事情

2、异步：客户端不需要等待服务器响应方，在服务器发出请求的时候，客户端可以进行其他操作。

实现方式：

1、原生的JS实现方式

JQeury实现方式 记住引入jQuery(并且重新build artifacts)

  1. $.ajax()

    语法：$.ajax({键值对});
    //使用$.ajax()发送异步请求

  1. $.post()：发送post请求

    • 语法：$.post(url, [data], [callback], [type])
    • 参数：
      • url：请求路径
        • data：请求参数
          • callback：回调函数
            • type：响应结果的类型

What Is Hexo?

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

官方文档: https://hexo.io/zh-cn/docs/index.html

Environment Requirements

安装Hexo只需要预先安装两个应用：

  • Node.js
    建议使用长期支持版本（LTS），以及版本不应该低于8.10。
  • Git
    配置Git可以查看Git提供的官方文档：https://git-scm.com/book/en/v2

Node.js

安装网址：https://nodejs.org/en/

推荐下载LTS版本，基本不会遇到什么配置问题。

其中需要提醒的是，不要安装一个额外Node.js 支持，这个资源很大且资源服务器在国外，因此会导致下载会很慢。

在安装Node.js之后请记住安装nodejs的路径，以及nodejs包管理工具npm的路径。

如果安装成功，在命令行窗口键入 node -v npm

Hexo中插入图片

在写文章时，常常有配图说明的需求。Hexo有多种图片插入方式，可以将图片存放在本地引用或者将图片放在CDN上引用。

本地引用

绝对路径

当Hexo项目中只用到少量图片时，可以将图片统一放在source/images文件夹中，通过markdown语法访问它们。

图片既可以在首页内容中访问到，也可以在文章正文中访问到。

相对路径

图片除了可以放在统一的images文件夹中，还可以放在文章自己的目录中。文章的目录可以通过配置_config.yml来生成。

将_config.yml文件中的配置项post_asset_folder设为true后，执行命令$ hexo new post_name，在source/_posts中会生成文章post_name.md和同名文件夹post_name。将图片资源放在post_name中，文章就可以使用相对路径引用图片资源了。

上述是markdown的引用方式，图片只能在文章中显示，但无法在首页中正常显示。

如果希望图片在文章和首页中同时显示，可以使用标签插件语法。

CDN引用

除了在本地存储图片，还可以将图片上传到一些免费的CDN服务中。比如Cloudinary提供的图片CDN服务，在Cloudinary中上传图片后，会生成对应的url地址，将地址直接拿来引用即可。
