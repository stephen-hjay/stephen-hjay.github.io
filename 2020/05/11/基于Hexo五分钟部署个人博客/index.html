基于Hexo五分钟部署个人博客

Posted on 2020-05-11

What Is Hexo?

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

官方文档: https://hexo.io/zh-cn/docs/index.html

Environment Requirements

安装Hexo只需要预先安装两个应用：

  • Node.js
    建议使用长期支持版本（LTS），以及版本不应该低于8.10。
  • Git
    配置Git可以查看Git提供的官方文档：https://git-scm.com/book/en/v2

Node.js

安装网址：https://nodejs.org/en/

推荐下载LTS版本，基本不会遇到什么配置问题。

其中需要提醒的是，不要安装一个额外Node.js 支持，这个资源很大且资源服务器在国外，因此会导致下载会很慢。

在安装Node.js之后请记住安装nodejs的路径，以及nodejs包管理工具npm的路径。

如果安装成功，在命令行窗口键入 node -v npm

安装CNPM

由于镜像源都在国外，速度会很慢，所以需要用淘宝提供的镜像源提高下载速度。

同样的，可以使用cnpm -v 命令，看是否有返回版本号来查看是否成功安装。

本地预览

启动服务器
默认情况下，访问网址为：http://localhost:4000/。
hexo server// 也可以缩写为: hexo s
运行上面的命令后，打开浏览器输入http://localhost:4000/即可看到页面了，有木有很激动。

但是有很多时候，电脑的4000端口有可能会被占用。这种情况下，在进行访问的时候，浏览器就会一直无法加载成功。

这时候我们打开命令行，查看端口使用情况：

1 使用win+R组合键打开cmd。
2 输入命令netstat -aon|findstr "XXXX"可以查看某个端口的占用情况。在这里我们输入netstat -aon|findstr "4000"，结果如下：

表中的第二列是本地地址，最后一列是程序的PID。从中我们可以看出占用4000端口的程序PID是2204。

Hexo中插入图片

在写文章时，常常有配图说明的需求。Hexo有多种图片插入方式，可以将图片存放在本地引用或者将图片放在CDN上引用。

本地引用

绝对路径

当Hexo项目中只用到少量图片时，可以将图片统一放在source/images文件夹中，通过markdown语法访问它们。

图片既可以在首页内容中访问到，也可以在文章正文中访问到。

相对路径

图片除了可以放在统一的images文件夹中，还可以放在文章自己的目录中。文章的目录可以通过配置_config.yml来生成。

将_config.yml文件中的配置项post_asset_folder设为true后，执行命令$ hexo new post_name，在source/_posts中会生成文章post_name.md和同名文件夹post_name。将图片资源放在post_name中，文章就可以使用相对路径引用图片资源了。

上述是markdown的引用方式，图片只能在文章中显示，但无法在首页中正常显示。

如果希望图片在文章和首页中同时显示，可以使用标签插件语法。

CDN引用

除了在本地存储图片，还可以将图片上传到一些免费的CDN服务中。比如Cloudinary提供的图片CDN服务，在Cloudinary中上传图片后，会生成对应的url地址，将地址直接拿来引用即可。
